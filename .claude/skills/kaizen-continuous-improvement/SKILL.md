# Kaizen - Continuous Improvement

## When to use this skill

- Improving development processes incrementally
- Optimizing team workflows and practices
- Reducing waste and inefficiencies
- Enhancing code quality over time
- Implementing retrospective learnings
- Fostering continuous improvement culture
- Identifying process bottlenecks
- Streamlining development pipelines
- Improving code review processes
- Optimizing deployment workflows
- Reducing technical debt systematically
- Implementing small, incremental improvements

## When to use this skill

- Applying incremental improvements, retrospectives.
- When working on related tasks or features
- During development that requires this expertise

**Use when**: Applying incremental improvements, retrospectives.

## Process
1. Identify bottleneck
2. Propose small change
3. Implement quickly
4. Measure impact
5. Iterate
